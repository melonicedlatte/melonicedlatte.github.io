﻿git 을 이용하다가 파일 이름이 변경되거나 삭제가 되면, 
git add * 명령어를 이용해도 서버에 반영이 되지 않습니다. 

아래와 같이 나오게 됩니다. 


커밋하도록 정하지 않은 변경 사항:
  (무엇을 커밋할지 바꾸려면 "git add/rm <파일>..."을 사용하십시오)
  (작업 폴더의 변경 사항을 버리려면 "git checkout -- <파일>..."을 사용하십시오)
        삭제함:        "\355\205\220\354\204\234\355\224\214\353\241\234\354\232\260 \355\225\231\354\212\265 1.ipynb"
        삭제함:        "\355\205\220\354\204\234\355\224\214\353\241\234\354\232\260 \355\225\231\354\212\265 2.ipynb"
        삭제함:        "\355\205\220\354\204\234\355\224\214\353\241\234\354\232\260 \355\225\231\354\212\265 3.ipynb"


불필요한 파일은 지워 주는게 좋기 때문에, 아래의 명령어를 사용합니다. (-u 옵션의 의미는 update tracked files)

git add -u


이제, git status 명령어를 통해 확인해보면, 아래와 같이 잘 반영이 됩니다. 



그리고 commit -m 을 하여 태그를 만들어주고, push 를 해주면 서버에도 잘 변경이 됩니다 !!
