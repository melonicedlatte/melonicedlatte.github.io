문제

총 8개의 톱니를 가지고 있는 톱니바퀴 4개가 아래 그림과 같이 일렬로 놓여져 있다. 또, 톱니는 N극 또는 S극 중 하나를 나타내고 있다. 톱니바퀴에는 번호가 매겨져 있는데, 가장 왼쪽 톱니바퀴가 1번, 그 오른쪽은 2번, 그 오른쪽은 3번, 가장 오른쪽 톱니바퀴는 4번이다.

이 때, 톱니바퀴를 총 K번 회전시키려고 한다. 톱니바퀴의 회전은 한 칸을 기준으로 한다. 회전은 시계 방향과 반시계 방향이 있고, 아래 그림과 같이 회전한다.

톱니바퀴를 회전시키려면, 회전시킬 톱니바퀴와 회전시킬 방향을 결정해야 한다. 톱니바퀴가 회전할 때, 서로 맞닿은 극에 따라서 옆에 있는 톱니바퀴를 회전시킬 수도 있고, 회전시키지 않을 수도 있다. 톱니바퀴 A를 회전할 때, 그 옆에 있는 톱니바퀴 B와 서로 맞닿은 톱니의 극이 다르다면, B는 A가 회전한 방향과 반대방향으로 회전하게 된다. 예를 들어, 아래와 같은 경우를 살펴보자.

두 톱니바퀴의 맞닿은 부분은 초록색 점선으로 묶여있는 부분이다. 여기서, 3번 톱니바퀴를 반시계 방향으로 회전했다면, 4번 톱니바퀴는 시계 방향으로 회전하게 된다. 2번 톱니바퀴는 맞닿은 부분이 S극으로 서로 같기 때문에, 회전하지 않게 되고, 1번 톱니바퀴는 2번이 회전하지 않았기 때문에, 회전하지 않게 된다. 따라서, 아래 그림과 같은 모양을 만들게 된다.

위와 같은 상태에서 1번 톱니바퀴를 시계 방향으로 회전시키면, 2번 톱니바퀴가 반시계 방향으로 회전하게 되고, 2번이 회전하기 때문에, 3번도 동시에 시계 방향으로 회전하게 된다. 4번은 3번이 회전하지만, 맞닿은 극이 같기 때문에 회전하지 않는다. 따라서, 아래와 같은 상태가 된다.

톱니바퀴의 초기 상태와 톱니바퀴를 회전시킨 방법이 주어졌을 때, 최종 톱니바퀴의 상태를 구하는 프로그램을 작성하시오.

입력

첫째 줄에 1번 톱니바퀴의 상태, 둘째 줄에 2번 톱니바퀴의 상태, 셋째 줄에 3번 톱니바퀴의 상태, 넷째 줄에 4번 톱니바퀴의 상태가 주어진다. 상태는 8개의 정수로 이루어져 있고, 12시방향부터 시계방향 순서대로 주어진다. N극은 0, S극은 1로 나타나있다.

다섯째 줄에는 회전 횟수 K(1 ≤ K ≤ 100)가 주어진다. 다음 K개 줄에는 회전시킨 방법이 순서대로 주어진다. 각 방법은 두 개의 정수로 이루어져 있고, 첫 번째 정수는 회전시킨 톱니바퀴의 번호, 두 번째 정수는 방향이다. 방향이 1인 경우는 시계 방향이고, -1인 경우는 반시계 방향이다.

출력

총 K번 회전시킨 이후에 네 톱니바퀴의 점수의 합을 출력한다. 점수란 다음과 같이 계산한다.

  • 1번 톱니바퀴의 12시방향이 N극이면 0점, S극이면 1점
  • 2번 톱니바퀴의 12시방향이 N극이면 0점, S극이면 2점
  • 3번 톱니바퀴의 12시방향이 N극이면 0점, S극이면 4점
  • 4번 톱니바퀴의 12시방향이 N극이면 0점, S극이면 8점

>> 문제 풀이

전형적인 삼성 기출 문제의 느낌이다. 
알고리즘의 복잡도보다는 구현 자체가 굉장히 복잡하다. 

최대한 메소드를 쪼개서 간결하게 하려고 노력했다.
톱니의 숫자는 배열을 통해서 받아냈다. 

중간에 깨달은 것은 톱니를 돌려야 하는 것을 처음에 체크를 한 다음에, 한 번에 돌리는 것이 더 효율적이라는 것이다. 

처음에 돌려버리면 처리가 까다로워진다. 

pair <int, int> 를 통해서 첫 번째 인자는 from, 두 번째 인자는 to 로 설정했다. 
n부터 시작하면 인접한 n-1, n+1를 queue에 넣어준다. {n, n-1} , {n, n+1}
이 때, n+1, n-1는 0과 3 사이의 숫자여야 한다.
아닌 경우 queue에 넣지 않는다. 

from 과 to 에서 작은 숫자의 2번 index와 큰 숫자의 6번 index를 비교하여 일치 여부를 살피고,
회전 배열은 rotateWay[] 이고, 회전을 하는 경우에는 from * (-1)
즉, from 의 반대 방향으로 회전 값을 넣어준다. 


상세한 프로세스는 주석을 통해서 전개했다. 

>> 느낀 점 

queue 자료구조를 쓰고, pair 을 안에 넣으려고 하는 풀이 방식을 처음에 생각하지는 못했다.
문제에 있어서 설계가 중요하지만, 100% 설계를 하고 들어갈 수는 없는 것 같다.

문제를 읽으면서 깨달아 가는 점도 있기 때문이다. 

중간에 c언어 배열 인덱스 설정을 또 실수한 적이 있었다. 
배열을 어떻게 하면 손쉽게 다룰 수 있을까??

#include <iostream>
#include <queue>
#include <math.h>
 
#define TOBNI_NUM 4
#define TIME 8
 
//#pragma warning(disable:4996)
using namespace std; 
 
// 0 이 12시 나머지는 자신의 번호대로 넘버 
int tobni[TOBNI_NUM][TIME] = {0};
int way[2] = { -1, 1 };
int K;
 
// 12시에 있는 값들을 출력
int print_answer() {
    int answer = 0;
    for (int i = 0; i < TOBNI_NUM; i++)
        answer += (tobni[i][0] == 1 ? pow(2, i) : 0 );
 
    return answer;
}
 
// n번째의 회전하는 톱니 숫자를 변경해준다. 
void change_tobni_num(int rotate_tobni, int rotate_way) {
    // 시계
    if (rotate_way == 1) {
        int sub = tobni[rotate_tobni][TIME-1];
        for (int i = TIME-1; i > 0; i--) {
            tobni[rotate_tobni][i] = tobni[rotate_tobni][i-1]; 
        }
        tobni[rotate_tobni][0] = sub;
    }
    // 반시계
    else if(rotate_way == -1) {
        int sub = tobni[rotate_tobni][0];
        for (int i = 1; i  < TIME; i++) {
            tobni[rotate_tobni][i-1] = tobni[rotate_tobni][i];
        }
        tobni[rotate_tobni][TIME - 1] = sub;
    }
}
 
// 톱니를 회전한다. 
void rotate_tobni(int rotate_tobni, int rotate_way) {
    // 0 은 회전 안한 것. 1 은 시계방향, -1은 반시계 방향 
    int rotateWay[TOBNI_NUM] = { 0, 0 , 0 , 0 };
 
    // 회전 체크
    rotateWay[rotate_tobni] = rotate_way;
 
    // 파생 되는 친구들을 비교하고 자석 극을 교체한다. 
    // pair<int , int>는  from -> to 
    queue<pair<int, int>> search_queue;
 
    // 인접한 친구 중에 가능한 친구 확인 해서 queue에 넣기 
    int start = rotate_tobni;
    for (int i = 0; i < 2; i++) {
        int next = start + way[i];
        if ( (next >= 0) && (next < TOBNI_NUM) && (rotateWay[next] == 0) && (rotateWay[start] != 0) )
            search_queue.push({ start, next });
    }
    
    while (search_queue.size()) {
        pair<int, int> move_pair = search_queue.front();
        search_queue.pop();
 
        // 옆자리 확인 ( 적은 숫자는 왼쪽 톱니[2], 큰 숫자는 오른쪽 톱니[6] )
        int small = (move_pair.first < move_pair.second ? move_pair.first : move_pair.second);
        int big = (move_pair.first < move_pair.second ? move_pair.second : move_pair.first);        
 
        // 같으면 안움직이고 끝. 
        if (tobni[small][2] == tobni[big][6]) continue;
 
        // 다르면 안 움직인거 회전 배열에 표시
        rotateWay[move_pair.second] = -(rotateWay[move_pair.first]);
 
        // 인접한 친구 중에 가능한 친구 확인 해서 queue에 넣기 
        int start = move_pair.second;
        for (int i = 0; i < 2; i++) {
            int next = start + way[i];
            if ((next >= 0) && (next < TOBNI_NUM) && (rotateWay[next] == 0) && (rotateWay[start] != 0))
                search_queue.push({ start, next });
        }
 
    }
 
    // 회전 배열에 맞춰서 한 번에 회전한다. 
    for (int i = 0; i < TOBNI_NUM; i++) {
        change_tobni_num(i, rotateWay[i]);
    }
}
 
int main() {
    // 톱니 초기화
    for (int i = 0; i < TOBNI_NUM; i++)
        for (int k = 0; k < TIME; k++)
            scanf("%1d",  &tobni[i][k]);
    
    // 회전 횟수 받기
    cin >> K;
    for (int i = 0; i < K; i++) {
        int tobni_num, rotate_way;
        cin >> tobni_num  >> rotate_way;
        // 톱니 회전 
        rotate_tobni(tobni_num-1, rotate_way);
    }
 
    cout << print_answer();
 
    return 0;
}
