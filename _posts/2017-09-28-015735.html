파일 질라를 통해 원격으로 파일을 주고 받으려고 하는데 자꾸 에러가 나더라구요 
그래서 블로그 호스팅 서버의 ftp 포트를 보니 잘 작동하고 있더라구요. ( 21번 포트로요 ) 


>> 에러 메세지 1

상태:	연결 수립, 환영 메시지를 기다림...
상태:	보안되지 않은 서버입니다. TLS를 통한 FTP를 지원하지 않습니다.
상태:	서버에서 비 ASCII 문자를 지원하지 않습니다.
상태:	로그인
상태:	디렉터리 목록 조회...
상태:	서버가 알 수 없는 주소와 수동형 응답을 보냈습니다. 대신 서버 주소를 사용합니다.
상태:	000.000.000.000:21에 연결... ( 서버 IP )상태:	연결 수립, 환영 메시지를 기다림...
상태:	보안되지 않은 서버입니다. TLS를 통한 FTP를 지원하지 않습니다.
명령:	USER root응답:	331 Please specify the password.
명령:	PASS ***********응답:	530 Login incorrect.
오류:	치명적 오류: 서버에 연결하지 못함

>> 에러 메세지 2

상태:	서버와의 연결이 종료됨상태:	000.000.000.000에 연결...
상태:	연결 수립, 환영 메시지를 기다림...
상태:	보안되지 않은 서버입니다. TLS를 통한 FTP를 지원하지 않습니다.
상태:	로그인상태:	디렉터리 목록 조회...
상태:	서버가 알 수 없는 주소와 수동형 응답을 보냈습니다. 대신 서버 주소를 사용합니다.
명령:	LIST오류:	Connection timed out after 20 seconds of inactivity
오류:	디렉터리 목록 조회 실패


저 같은 경우에는 위 2개의 메세지가 떴습니다. 
에러 1 은 root 계정, 에러2는 새로 생성한 test 계정을 통해서 들어갔습니다. 

두 개 다 비밀번호는 틀리지 않았는데 접속이 되지 않더라구요. 

그래서 메뉴 -> 사이트 관리자로 들어가서 프로토콜을 ftp -> sftp 로 바꾸어 주었습니다. 








이렇게 바꾸고 나서 접속하니 root 계정과 새로 생성한 test 계정 모두 ftp 에 성공했습니다. 

혹시 ftp 가 안되면 sftp 로 바꿔보세요 !!!
